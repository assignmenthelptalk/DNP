## Core Concept Def initions — Use These Exactly

### Topical Authority
Ranking higher for a certain amount of time compared to other web sources with higher
clarity, accuracy, information quality, and responsiveness for a cheaper search engine
evaluation and retrieval process, so that the web source is preferred by the search engine
for certain topics for prioritized rankings.

**Formula: Topical Authority = Topical Coverage × Historical Data**

This is NOT "focus on topics instead of keywords."
This is NOT "cover more topics."
Topical Authority is a STATE achieved through this exact formula.

Koray Tuğberk GÜBÜR founded this concept on 18 May 2022.
Proven result: 0 to 128,000 organic traffic in 123 days, achieved through
Semantic SEO only — without backlinks, technical SEO, or page speed optimization.

### Historical Data
NOT time. NOT domain age.

Historical Data = the amount and quality of user engagement for a certain period.
Includes: hover-overs, impressions, rankings at position 94, clicks, touches,
text-selects, clickless searches, query session logs.

Critical rules:
- Fresh historical data is more important than older historical data
- Non-quality engagement (poor session logs) demotes a website after time
- If rankings are lost today, the cause is historical data from at least 6 months ago
- Cleaning bad historical data requires building stronger positive signals to replace it
- A website with 2 years of history and 10M sessions has stronger historical data than
  a website with 10 years of history and 1 session

### Topical Coverage
Cannot be increased by:
- Stuffing entities
- Stuffing attributes
- Opening a new page for every topic

Topical Coverage requires creating a TOPICAL BORDER first to calculate it.

Coverage rules:
- If you didn't DEFINE X → you didn't cover it
- If you didn't CONNECT X to Y → you didn't cover it
- If you didn't match the MACRO-CONTEXT of the document to the QUERY CONTEXT → you didn't cover it
- If your definition misses certain aspects → you didn't cover it

Example depth required for "Electric Scooters" battery coverage:
Chemistry types (LiFePO4, NCA, NMC, LTO, LiPo, NiFe, Zn-Air, NaNiCl),
voltage types (operating, charging, peak, nominal, display, motor),
durability factors (vibration, water, shock, overcharging, cycle life, humidity, temperature),
AND all attributes must be connected back to each other AND to the electric scooter context.
Mentioning "battery quality" is not coverage. Specifying the above is coverage.

### Topical Map
A content network design based on semantics for achieving the topical authority state
by including overall macro contexts and semantics with publication frequency.

Has exactly two sections: Core Section and Outer Section.

This is NOT:
- Connections between entities via attributes
- A list of blog post names
- "Topics in SEO" or "content planning"

The phrase "Topical Map" was coined first by Koray Tuğberk GÜBÜR.

### Source Context
The purpose of the brand identity and how the brand monetizes its content.
Represents the main focus of the website and its connection to the central entity.
Must be reflected in site-wide N-grams, boilerplate, and main content on every page.

Example: Two sites covering "country" entities —
- Visa Consultancy: connects climate → best time to apply for visa, entry requirements
- Real Estate Consultancy: connects climate → investment conditions, property types
Same entity. Different Source Context. Completely different topical map.

### Central Entity
The entity that appears in every sub-section of the semantic content network —
in both main content/macro context AND supplementary content/micro context.
Gives its main attributes to the Core Section and minor attributes to the Outer Section.
Always appears in anchor texts with a synonym value.

### Central Search Intent
The unification of Source Context with the Central Entity.
Processed in both Core and Outer Sections of the topical map.
Connected to the macro-context and micro-context of every content brief.
Appears in boilerplate, main content, and supplementary content site-wide.

### Core Section of the Topical Map
The unification of Source Context with the Central Search Intent.
Focuses on the MAIN ATTRIBUTE of the central entity — the attribute that connects
directly to the source context.

Examples by source context:
- Electric car charger AFFILIATE → main attribute = "quality"; derived = durability, charge time, maintenance
- Electric car charger ENGINEER → main attribute = "production"; derived = materials, designs, types
- Visa Consultancy → main attribute = "visa" under country context
- Divorce Lawyer → main attributes = "divorce" (mediation, trial, alimony, custody) + "legal" (grounds, prenuptial, annulment)
- Pension Planning → main attribute = "retirement" under "financial independence" context

### Outer Section of the Topical Map
Focuses on MINOR attributes of the central entity.
Purpose: improve overall historical data, increase topical relevance and contextual
consolidation, propagate trust and quality signals TO the Core Section via links
or linkless connections.

Examples:
- Visa Consultancy Outer: country culture, religion, geography, climate, language schools
- Divorce Lawyer Outer: marriage statistics, relationship mistakes, marriage types,
  marriage dynamics, marital roles, marriage functions
- Pension Planning Outer: Elder Law Attorney, Power of Attorney, Estate Planning,
  Gifting, Annuity, Medicaid, Retirement Community

### Main Content
The main part of the web document.
- Processes specific context-terms and topical entries
- Gives the main context-flow and coverage including a summary of the entire article
- Satisfies MAJOR query needs
- Touches minor query needs ONLY at the Contextual Border — not throughout
- Processes the macro-context of the document
- Provides proper connection to query contexts and semantics
- Contains minimal internal links (only if inside the macro-context)
- Does NOT touch sub-contexts or minor topics

### Supplementary Content
The supplementary part of the web document.
- Touches micro-contexts, sub-topics, minor entities and attributes
- Always connected to the macro-context of the page
- Always processes topics WITH a connection to ANOTHER macro-context
- Provides better association and "Neighborhood Content" between topical map segments
- Contains the majority of internal links and contextual bridges

NOT: supporting content for the main topic.
NOT: a section added for completeness.
It has a specific structural role — bridging macro-context to other topical map nodes.

### Contextual Coverage
The amount of web page section for a specific context compared to other sections.
- Over-processing one context dilutes prominence of others
- Under-processing a context decreases relevance
- Connected to Passage Indexing (Ranking)
- Must not break the Contextual Vector

### Contextual Flow
The ORDER in which contextual vectors are processed in a web document.
The same information in a different order creates different click satisfaction scores.
Heading formats, words, and hierarchy change context priority.
Every ordering decision has a semantic consequence.

### Contextual Hierarchy
Determines which contextual vector comes before which.
Represented by typography, visuals, and coverage of specific sub-sections.
Changes the weight of internal links for relevance and PageRank pass.
Changes the macro-context of the web page.

### Contextual Border
The border between macro and micro context sections.
Provides a SLOW TRANSITION from main content to supplementary content.
Uses a grouper question to deepen the main context while opening to side-topics.

### Contextual Bridge
Hypertext or linkless connections in a semantic content network.
- Linkless: provided by aligning consistent information across documents without a link
- Linked: hypertext providing contextual connection between two different topical map nodes
- Focuses on query paths, sequences, and correlative queries

### Macro Semantics
Semantics for the main and prominent parts of a document's relevance and context.
Sources: headings, anchor texts, site-wide N-grams.

Macro semantics asks:
- What site-wide N-grams can appear on this website?
- What are the most used nouns, adjectives, predicates site-wide?
- What are the most used question formats site-wide?
- How are heading vectors constructed?
- What are the first words of paragraphs — do they answer or use rhetoric without information?
- What are the heaviest context terms?
- Do context terms match the query context?

### Micro Semantics
Semantics for micro-adjustments to context and relevance.
Sources: single word changes, punctuation, word-order.

Includes:
- Sequence modeling: word-by-word optimization for relevance and responsiveness
- Discourse integration optimization
- Sequence-to-sequence connections for changing contextualization
- Visual optimization with subject entities (most prominent object) and object entities
  (context-interpreting elements: "a fish and bear" vs "a lake and a bear")

Example: "Teacher yelled students" distributes relevance with "teacher" as heavier.
"Students are yelled by Teacher" distributes relevance with "students" as heavier.
The predicate "yell" annotates a different context from "shout" and changes all
surrounding word sequence probabilities.

### Relevance
The Information Retrieval Score from text processing methodologies:
term saturation, length normalization, co-occurrence matrix, BM25, TF-IDF, GloVe, Word2Vec.
Shows overall connection. The "Blind Librarian" state of the search engine.

### Responsiveness
The Information Extraction Process of the search engine for giving a direct answer.
Even without relevance, there may be a quality answer.
Requires query-question-answer pairing and indexing.
Connected to Passage Indexing (Ranking).

Relevance ≠ Responsiveness. Both must be optimized — separately.

### Represented vs Representative Queries
- Represented Query: exact terms the user types
- Representative Query: how the search engine actually interprets it

Search engines do NOT give full relevance weight to search bar terms.
Relevance is distributed based on representative queries primarily.

Example: "board vision" and "vision board" produce different SERPs because the search
engine creates different contextual connections and query interpretations for each.

For single-word query "cancer": representative queries include "learn cancer,"
"cancer treatment," "compare treatment methods," "cancer treatment center comparison."
The web document must satisfy all possible interpretations.

### Contextual Relevance
Contextual appropriateness and significance of terms for the given context.
Example: "love" can carry a sexual or religious context depending on surrounding terms.

### Vastness-Depth-Momentum
Simplification of everything in a semantic content network and topical map:
- Vastness = go wider (bigger topical map, more nodes)
- Depth = go deeper (more complete EAV per node, more attribute coverage)
- Momentum = go faster (higher publication frequency)

If one is missing, increase another to compensate.
A small topical map can succeed with extreme depth and momentum.
A wide map with low depth needs very high momentum.

### Publication Frequency / Momentum
Frequency of major content updates or new content publications.
Purpose: be prioritized for crawling rate, indexation, and rankings.
Takes the attention of the search engine's content exploration systems.

### Content Configuration
Process of optimizing relevance and responsiveness continuously according to changed
semantic distances and similarities of query terms.
Comes from changed query semantics after Broad Core Algorithm Updates (BCAUs).
Requires refreshing and reconfiguring the topical maps.
If the source is already topical authority, configuration is mostly needed from the
competitor side — your documents are already well-structured.
