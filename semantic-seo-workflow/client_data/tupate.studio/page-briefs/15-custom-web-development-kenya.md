### H2: Technology Stack Options for Custom Web Applications in Kenya

**Contextual directive:** Cover the technology choices available for custom Kenyan web applications — framed for a business owner who needs to understand the implications, not master the tech.

**EAV depth required:**
- Backend options:
  - PHP/Laravel: most common for Kenya-deployed applications — extensive developer pool locally, excellent for rapid development, proven on Kenyan hosting infrastructure
  - Node.js: real-time applications (live chat, live availability, notifications), strong community, growing Kenya developer adoption
  - Python/Django: data-heavy applications, financial systems, analytics dashboards
- Frontend options:
  - React.js: complex interactive user interfaces (real-time dashboards, single-page applications)
  - Vue.js: lighter alternative to React — suitable for medium-complexity Kenya business portals
  - Vanilla HTML/CSS/JS: for performance-critical Kenya websites where framework overhead would hurt mobile load times
- Database options:
  - MySQL/MariaDB: most common for Kenyan deployments, supported by all local hosting providers
  - PostgreSQL: for complex data relationships (financial systems, multi-tenant platforms)
- Stack selection criteria for Kenya: local developer availability for ongoing maintenance, hosting environment support, performance on Kenyan mobile networks

**Word count target:** 300–360 words

### H2: M-Pesa API and Kenyan Payment Gateway Integration

**Contextual directive:** Cover the M-Pesa Daraja API integration in depth — the most unique and most requested custom development feature for Kenyan businesses.

**EAV depth required:**
- Daraja API overview: Safaricom's developer API that enables businesses to programmatically initiate and receive M-Pesa payments
- API types relevant to custom web development:
  - STK Push (Lipa Na M-Pesa Online/M-Pesa Express): prompts user's phone to enter PIN for payment — best UX for online checkout in Kenya
  - C2B (Customer to Business): customer pays to a Paybill/Till, system receives confirmation callback
  - B2C (Business to Customer): business sends money to customer phone (refunds, employee payments, distributor payouts)
  - B2B: business-to-business transfers
- Integration process: Safaricom developer portal (developer.safaricom.co.ke) → sandbox credentials → develop → test → go-live application (requires registered M-Pesa business account)
- Callback URL requirement: your web application must have a publicly accessible HTTPS endpoint to receive payment confirmation from Safaricom servers
- Other Kenya payment gateways in custom development: Pesapal (M-Pesa + cards), iPay Africa, Flutterwave Kenya (cards + M-Pesa), Equity Bank PayStar

**Word count target:** 300–360 words

**Contextual directive:** Cover booking system development specifically for Kenyan business contexts — the most common custom development request for service businesses.

**EAV depth required:**

- Kenyan business use cases:
  - Clinic appointment booking (WhatsApp alternative — many Kenyan clinics still rely on WhatsApp DMs)
  - Hotel and lodge room booking (direct booking engine to reduce OTA commission dependency)
  - Car hire / equipment rental (availability management by vehicle/equipment unit)
  - Professional service booking (consultant/lawyer time slot scheduling)
  - Training centre / gym class booking
- SMS integration (Africa's Talking): booking confirmation SMS to Kenyan mobile number — critical as email open rates are low in Kenya for transactional messages
- Kenya-specific: booking confirmation via WhatsApp Business API — increasingly expected by Kenyan service clients

### H2: Hosting and Deployment for Custom Web Applications in Kenya

**Contextual directive:** Cover hosting requirements for custom web applications vs standard websites — VPS, cloud options, and Kenyan deployment considerations.

**EAV depth required:**
- Hosting type differences: custom web applications need VPS or cloud hosting — shared hosting lacks the control and resources needed for application server processes, cron jobs, database optimization
- VPS options for Kenya deployments:
  - DigitalOcean (closest Africa region: Amsterdam/Singapore with Cloudflare for Kenya CDN): reliable, scalable, developer-friendly
  - Linode (Akamai Cloud): good global CDN coverage, African presence
  - AWS ap-southeast (with CloudFront): for high-traffic applications, premium cost
  - Google Cloud africa-south1 (Johannesburg): lowest latency to Kenyan users among major cloud providers
  - Local VPS: Truehost Kenya VPS — affordable, local support, good for medium-complexity applications
- Deployment process: Git-based deployment (GitHub Actions or Bitbucket Pipelines), staging environment for testing before production, automated database backups, SSL/HTTPS configuration
- DevOps for Kenya applications: server monitoring (Uptime Robot free tier), error logging (Sentry), performance monitoring (New Relic or self-hosted Prometheus)

**Word count target:** 280–340 words
