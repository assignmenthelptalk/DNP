## Sequencing Logic

**Phase 1 — Core Tier 1 (Project Section Service Pages)**
Establish the five primary money pages — the main service overview plus one dedicated page per IOSH Managing Safely project section (hazard identification, risk assessment, action plan, hierarchy of control). These are the highest-conversion pages in the site and must be indexed before the marking criteria depth pages and industry-specific guidance launch. The project section pages are published in the order candidates encounter them when writing the project: hazard identification first, then risk assessment, then control measures, then action plan.

**Phase 2 — Core Tier 2A (Marking Intelligence + Test Preparation + Referral Recovery)**
Marking criteria, high-score strategy, test preparation, and referral and resubmission guide go live immediately after Phase 1. These capture the three highest-converting IOSH audiences: candidates about to submit their project (marking criteria and high-score strategy), candidates preparing for the test (test preparation), and candidates whose project has been failed (referral guide — May–June is the peak referral window for cohorts who sat courses in March–April).

**Phase 3 — Core Tier 2B (Industry-Specific Guidance + Gap Assets)**
Industry-specific project guidance and the two highest-priority gap assets (worked examples and project template) are published as a cluster. Publishing them after the project section pages are indexed maximises internal link density — every section page can link to the relevant industry example. The office/DSE worked example is published first as it serves the highest-volume candidate segment.

**Phase 4 — Core Tier 2C (Supporting Gap Topics)**
International candidates gap page, IOSH SHE Technician assessment help, and the NEBOSH full comparison gap page extend the Core Section to underserved audience segments before the September new intake.

**Phase 5 — Outer Tier 1 (Qualification Discovery and Comparison)**
IOSH overview, IOSH vs NEBOSH, Working Safely vs Managing Safely, online delivery comparison, and pass rates pages — published from late July through August targeting prospective candidates researching before September employer-mandated course enrolments.

**Phase 6 — Outer Tier 2 and 3 + Remaining Gap Topics**
Academic integrity, UK H&S legislation, incident investigation module, manager context, membership grades, careers, and foundational knowledge pages closing the content network.

## Internal Linking Priority

When publishing each page, apply these linking rules immediately:

- Every Core Tier 1 project section page links to: IOSH Assignment Help (main), IOSH Managing Safely Project Help (hub), and the adjacent project sections in the writing sequence (Hazard Identification → Risk Assessment → Hierarchy of Control → Action Plan)
- IOSH Managing Safely Project Help links to: all five project section pages — it is the primary internal linking hub and every section page must receive a link from it on publication
- IOSH Managing Safely Project Marking Criteria links to: all five project section pages (marks each section) and the How to Achieve a High Score page — these three pages form a marking intelligence cluster
- IOSH Managing Safely Referral and Resubmission Guide links to: Hazard Identification Help and Risk Assessment Help (the two most common referral reasons), Project Help hub, and Marking Criteria
- IOSH Managing Safely Test Preparation links to: UK H&S Law page and Incident Investigation page (once live) — test revision cluster
- Every industry-specific and worked example page links to: IOSH Managing Safely Hazard Identification Help, IOSH Managing Safely Risk Assessment Help, IOSH Hierarchy of Control page, and IOSH Managing Safely Project Action Plan Help
- IOSH Managing Safely Project Template links to: all five Core Tier 1 project section pages — the template page is the highest-conversion page and must direct candidates to all section guidance immediately
- Every Outer Tier 1 page links to: IOSH Managing Safely Project Help and at least one Core Tier 1 project section page (see topical_map.json contextual_bridges for exact anchor texts)
