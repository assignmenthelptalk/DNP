## New H2 Section Specification

### H2: BTEC IT Distinction: Evaluating Technical Solutions Against Standards and Alternatives
> **EAV to cover**: Domain-specific Distinction evaluation frameworks — what evaluation means in Programming, Networking, Databases, and Systems Analysis; specific standards and frameworks by domain
> **Word count**: 300–360 words
> **Specific content**:
> - **Programming**: Distinction requires algorithmic complexity evaluation using Big O notation at HNC/HND — not just implementing a working solution but evaluating whether the chosen algorithm's time and space complexity is appropriate for the data scale, and comparing against alternative algorithms with justified reasoning. SOLID principles assessment (Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion) provides the evaluation framework for OOP design quality at HNC level
> - **Networking**: Distinction requires systematic security evaluation — identifying the threats the network faces (using a recognised threat taxonomy such as STRIDE: Spoofing, Tampering, Repudiation, Information Disclosure, Denial of Service, Elevation of Privilege), evaluating how each security control in the design mitigates the identified threats, and assessing residual vulnerabilities with justified remediation. Alignment with the NCSC Cyber Essentials scheme (five controls: firewalls, secure configuration, access control, malware protection, patch management) provides a structured evaluation framework accepted at HNC and HND level
> - **Databases**: Distinction requires comparative DBMS evaluation — not just implementing a database solution but justifying the choice of database management system (MySQL vs PostgreSQL vs Microsoft SQL Server vs NoSQL approaches) against the specific requirements, using performance, scalability, cost, and licensing criteria. Evaluating when a NoSQL database (document, key-value, graph) would be more appropriate than a relational model, with reference to CAP theorem trade-offs, demonstrates the theoretical depth Distinction requires at HNC/HND
> - **Systems Analysis**: Distinction requires SDLC methodology evaluation — comparing waterfall, agile (Scrum/Kanban), and spiral methodologies against the specific characteristics of the project being analysed, justifying the recommended approach with reference to project scale, requirements stability, risk profile, and client involvement expectations. IEEE standards (IEEE 830 for software requirements, IEEE 1016 for software design) provide the authoritative framework for systems analysis evaluation at HNC level
