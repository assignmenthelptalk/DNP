## EAV Requirements

### Entity: Learning Technology Standards
- SCORM (Sharable Content Object Reference Model): packages e-learning content for LMS deployment; tracks completion and quiz scores; widely used but limited to basic data
- xAPI (Experience API / Tin Can): tracks any learning experience — on-the-job, mobile, simulation, social; sends data to Learning Record Store (LRS); enables rich learning analytics
- LMS: hosts, delivers, and tracks formal e-learning; SCORM-compatible; common examples: Moodle, Cornerstone, SAP SuccessFactors
- LXP (Learning Experience Platform): curates personalised learning paths, social features, informal content; common examples: Degreed, EdCast, LinkedIn Learning

### Entity: Mayer's Multimedia Learning Principles (key 5 for assignment use)
- Coherence: remove irrelevant material — less is more
- Signalling: highlight key information with cues (colour, arrows, bold)
- Redundancy: do not read full text aloud over slides — use narration or text, not both
- Spatial contiguity: place text near the image it describes
- Temporal contiguity: present narration and animation simultaneously, not sequentially

### Entity: Cognitive Load Theory (Sweller)
- Intrinsic load: complexity of the content itself — cannot be reduced
- Extraneous load: complexity caused by poor design — can and must be reduced
- Germane load: mental effort that builds new schemas — what good instructional design promotes
