## EAV Depth Map
**Primary Entity**: CMI management report

**Main Attributes**:
- Report structure (title page, executive summary, contents, introduction, main body sections, conclusion, recommendations, bibliography, appendices)
- Command verbs triggering report format (Evaluate, Justify, Assess, Recommend, Develop a plan)
- Word count range (2,500–5,000 words typical for Level 5; up to 6,000+ for Level 7)
- Levels where report is primary format (Level 5: most common; Level 6: standard; Level 7: strategic reports/papers)
- Harvard referencing (in-text citations + bibliography, minimum 10–15 sources for Level 5+)
- Tone (professional, third-person, evidence-based, practitioner-academic blend)

**Derived Attributes**:
- Executive summary writing (standalone summary of report — often marked separately)
- Recommendations section (must be actionable, specific, SMART where appropriate)
- Workplace application (CMI reports must connect theory to the student's real organisation or a case study)

**Values per Attribute**:
- Level 5 report → 3,000–5,000 words → executive summary 250–300 words → 10+ sources → Evaluate/Justify command verbs
- Level 7 report → 4,000–6,000 words → strategic analysis sections → Porter's Five Forces, PESTLE, SWOT may be required → 15+ sources
- Executive summary → written last, summarises key findings and recommendations → no more than 10% of total word count
- Recommendations → minimum 3 specific, justified recommendations per unit → linked to theory applied in main body

**Attributes on THIS page**: Report structure, command verbs for reports, executive summary, recommendations, word count by level, writer credentials, ordering process.

**Attribute depth warning**: Do not discuss essay structure here — that dilutes the report macro-context. CMI has distinct formats and mixing them confuses students and search engines.

> **Context vector**: CMI management report — definition, format trigger, and level prevalence
> **Main entity + attributes to cover**:

##### H3: Writing the Executive Summary for a CMI Report
> **Attribute + Value**:
> - Executive summary → written AFTER main report → summarises key analysis points and recommendations → typically 250–300 words → self-contained → written in past tense → no references
> - Common mistake: writing it like an introduction (future tense, describing what the report "will" cover) instead of summarising what it found
> **Writer note**: This is a frequent student error that causes mark deductions. State explicitly that our writers write the executive summary as a true standalone summary, not a reworded introduction.

##### H3: Recommendations Section — How to Write Justified CMI Recommendations
> **Attribute + Value**:
> - Minimum 3 recommendations per report → each linked to theory discussed in main body → SMART framing (Specific, Measurable, Achievable, Relevant, Time-bound) → no recommendations without justification
> - Common error: listing generic recommendations without connecting to evidence or theory
> **Writer note**: Specifically name SMART here — CMI assessors frequently reference SMART in their marking rubrics. Writers must apply it and label it.
