## EAV Depth Map
**Primary Entity**: EBP Frameworks for DNP Capstone

**Main Attributes**: Iowa Model (stages), PARIHS (elements), ACE Star (stages), KTA (cycles), ARCC (components), Johns Hopkins (process), selection criteria, justification language

**Derived Attributes**:
- **Iowa Model (2017 Revised)**: trigger identification → assemble team → is it a priority → search and synthesise evidence → is evidence sufficient → pilot → implement → evaluate and sustain → disseminate. Two types of triggers: problem-focused (quality issue, clinical data) and knowledge-focused (new research, guidelines).
- **PARIHS/i-PARIHS**: Evidence (research evidence + clinical experience + patient values/preferences) × Context (organisational culture, leadership, evaluation systems) × Facilitation (purpose, role, skills of the facilitator). i-PARIHS 2015 revision adds the "i" for implementation context.
- **ACE Star Model**: 5 stages — Knowledge Discovery (primary research) → Evidence Summary (systematic review) → Translation (clinical guidelines) → Integration (practice change) → Evaluation (outcomes). DNP capstones primarily occupy stages 3–5.
- **KTA Framework (Knowledge-to-Action)**: inner cycle = knowledge creation (knowledge inquiry, synthesis, tools/products); outer cycle = action cycle (identify problem, adapt knowledge to local context, assess barriers, select interventions, implement, monitor, sustain). Developed by Graham et al. (2006).
- **ARCC Model (Advancing Research and Clinical Practice Through Close Collaboration)**: focuses on building an EBP culture and mentor capacity; more organisational/system-level than project-level; used in Nurse Executive capstones.
- **Johns Hopkins EBP Model**: PET Process (Practice question, Evidence, Translation); three-phase approach; evidence level and quality rating system (Level I–V); particularly used in hospital-based nursing practice settings.

#### H2: Why Your DNP Capstone Needs an EBP Framework
> Frameworks provide structural scaffolding for the entire capstone — they organise the literature review, guide the implementation plan, and justify the methodology. Committee members expect the framework to be named, described, and explicitly mapped to project activities in Chapter 2 and the final manuscript.

#### H2: Iowa Model of Evidence-Based Practice (2017 Revised): Step-by-Step
> Full stage description with mapping examples for DNP capstone. Most widely used in nursing QI and EBP projects. Trigger types. When to choose Iowa Model: when the project starts from a clinical problem or quality gap (most common DNP scenario).

#### H2: PARIHS and i-PARIHS Framework: Evidence, Context, and Facilitation
> Three elements and their sub-components. When to choose PARIHS: when the implementation challenge is primarily about organisational context, culture, or facilitation rather than evidence strength. Common in complex multi-stakeholder implementations. i-PARIHS 2015 revision adds implementation context as a fourth element.

#### H2: ACE Star Model of Knowledge Transformation: Five Stages Explained
> Five stages with DNP capstone positioning (stages 3–5). When to choose ACE Star: when the proposal narrative needs to frame the capstone as a step in a larger translational science process.

#### H2: KTA (Knowledge-to-Action) Framework
> Inner and outer cycles explained. When to choose KTA: when the implementation challenge involves adapting evidence to a specific local context (cultural, geographic, resource-constrained settings). Common in Population Health and international nursing contexts.

#### H2: ARCC Model and Johns Hopkins EBP Model
> Brief descriptions. ARCC: use for system/organisational-level EBP culture change — Nurse Executive capstones. Johns Hopkins PET: use when the clinical site already uses Johns Hopkins tools — provides consistency with institutional EBP infrastructure; Level I–V evidence rating system is institutionally recognised.

#### H2: How to Choose and Justify Your EBP Framework for a DNP Proposal
> Selection criteria: project type (QI vs EBP implementation), setting complexity, organisational fit, framework familiarity. Justification language template: "The [framework name] was selected for this project because [reason 1 — fit with project type], [reason 2 — alignment with clinical setting/culture], and [reason 3 — framework stages map directly to the project phases]. Specifically, [Stage X] of the framework corresponds to [Project Component Y]." One framework only — do not mix frameworks.

### FAQ SECTION

**Q: Which EBP framework is most commonly used in DNP capstone projects?**
The Iowa Model of Evidence-Based Practice (2017 revised version) is the most commonly used EBP framework in DNP capstone projects — particularly for quality improvement and EBP implementation projects in acute care and primary care settings. It is straightforward to apply, widely recognised in nursing literature, and maps clearly to the proposal chapter structure.

**Q: Can I use multiple EBP frameworks in my DNP capstone?**
No. DNP proposals should select one EBP framework and apply it consistently throughout the proposal and final manuscript. Using multiple frameworks creates analytical incoherence and suggests the student does not fully understand either framework. If a change management framework (Kotter, Lewin, Rogers) is also used, it should be presented as a complementary conceptual framework — not as an additional EBP framework.
