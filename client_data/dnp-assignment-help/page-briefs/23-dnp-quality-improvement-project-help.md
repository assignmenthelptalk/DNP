## EAV Depth Map
**Primary Entity**: DNP Quality Improvement Capstone Project

**Main Attributes**: QI methodology, PDSA structure, IHI Model, AIM statement, process/outcome measures, EHR data, IRB classification, SPC charts

**Derived Attributes**:
- **QI Methodology**: IHI Model for Improvement (three fundamental questions: What are we trying to accomplish? How will we know that a change is an improvement? What changes can we make that will result in improvement?) + PDSA cycle (Plan-Do-Study-Act — rapid iterative testing cycles); DMAIC (Define-Measure-Analyse-Improve-Control — Six Sigma framework, less common in DNP); Lean waste identification (TIMWOOD: Transportation, Inventory, Motion, Waiting, Overproduction, Overprocessing, Defects)
- **AIM Statement**: a specific, measurable goal statement for a QI project; components: What? By how much? By when? For whom? In what setting? Example: "By [date], increase the CAUTI bundle compliance rate from 65% to ≥90% among ICU nurses at [facility name] through implementation of a standardised CAUTI prevention education and audit protocol."
- **PDSA Cycle Structure**: Plan (identify the change, predict what will happen, plan data collection), Do (implement the change on a small scale), Study (compare data to prediction, document what was learned), Act (adopt, adapt, or abandon the change based on learning → start next PDSA cycle). Multiple PDSA cycles in one DNP capstone: education PDSA (Week 1–2), implementation PDSA (Week 3–8), sustainability PDSA (Week 9–12)
- **Process Measures**: adherence/compliance rates — what percentage of the time is the intervention being performed as intended? Tells you whether the change is being implemented. Examples: hand hygiene compliance rate (%), CAUTI bundle element completion rate (%), checklist documentation rate (%)
- **Outcome Measures**: the actual clinical result — what changed for patients? CAUTI rate per 1000 catheter days, fall rate per 1000 patient days, 30-day readmission rate (%)
- **EHR Data**: most hospital-based DNP QI projects use pre-existing EHR audit data — no new data collection required. Sources: infection control database (HAIs), NDNQI nursing quality indicators, EHR audit reports (documentation compliance), pharmacy dispensing data (medication errors). This data availability simplifies IRB classification (QI non-research)
- **IRB Classification**: almost always QI/exempt for hospital-acquired condition projects and process compliance projects — uses routinely collected, de-identified quality data; findings intended for local improvement only
- **SPC Chart Selection**: P-chart (for proportions: compliance rates, infection rates as %; requires constant denominator), U-chart (for rates: HAIs per 1000 device days; varying denominator), I-MR chart (for individual measurements: weekly mean pain scores), run chart (median-based; for N<20 data points; sufficient for most DNP QI run charts)

#### H2: What Is a DNP Quality Improvement Capstone Project?
> Define QI in the DNP context: systematic, data-driven effort to improve healthcare processes and outcomes using structured improvement methodology. Most common DNP project type — uses PDSA/IHI methods. Distinguishes from EBP: QI starts from a quality gap in existing data; EBP starts from a formal literature synthesis. QI does not require generating new knowledge — it applies known best practices to fix local implementation gaps.

#### H2: IHI Model for Improvement vs PDSA Cycle: Which Framework for Your QI Project?
> IHI Model: the strategic framework (three questions define the aim, measures, and change theory). PDSA: the tactical cycle (iterative rapid testing of changes). Relationship: PDSA cycles are the tactical tool within the IHI Model. Multiple PDSA cycles are run within one QI project — typically 2–3 cycles covering education, implementation, and sustainability. When to choose IHI Model vs pure PDSA: IHI Model provides the AIM statement and measurement framework; PDSA provides the implementation cycle structure.

#### H2: Writing an AIM Statement for Your DNP QI Project
> AIM statement structure: What? (specific change) + How much? (measurable target with baseline) + By when? (timeline) + For whom? (population) + In what setting? (specific unit). Strong example: "By [12 weeks after implementation begins], increase CAUTI bundle compliance rate from 62% to ≥90% among RNs and APRNs in the 20-bed medical ICU at [hospital name] through implementation of a standardised CAUTI prevention bundle education and weekly audit protocol." Weak example: "Improve patient safety in the ICU" — no baseline, no target, no timeline, no specific population or measure.

#### H2: Selecting QI Process and Outcome Measures That Committees Approve
> Process measures (implementation fidelity), outcome measures (clinical result), balancing measures (unintended consequences — e.g., does increasing catheter removal frequency cause any harm?). For each project topic, identify at least one of each. SMART criteria for measure selection.

#### H2: Using Existing EHR Data in Your DNP QI Project
> Most hospital-based QI projects use pre-existing EHR quality data — no new data collection instrument required. Sources: infection control surveillance data (NHSN), EHR audit reports, NDNQI quality indicator dashboards, pharmacy data, documentation compliance reports. How to request de-identified aggregate data from the quality department or informatics team. Why using existing data simplifies IRB (QI non-research determination).

#### H2: Statistical Process Control Charts for DNP Quality Improvement
> P-chart, U-chart, I-MR chart, run chart — selection guide with use cases and chart type rules. Minimum data points for SPC (20+ for control limits). Run chart as simpler alternative. Special cause vs common cause variation. How SPC charts appear in the results section of the manuscript.

#### H2: Does a DNP QI Project Need IRB Approval?
> 45 CFR 46 QI determination: most QI projects do NOT require full IRB review — they qualify as QI non-research. Criteria: routine data, local improvement intent, not generalizable knowledge. Process: submit QI determination request to IRB; receive determination letter; proceed to implementation. PMHNP and population health QI projects with vulnerable populations are exceptions.

### FAQ SECTION

**Q: What is the difference between a DNP quality improvement project and an EBP implementation project?**
A quality improvement project starts from a gap in local quality metrics — the evidence for the solution is already established, and the project aims to implement a known best practice that is underperforming at the site. An EBP implementation project starts from a formal literature synthesis that identifies an evidence-based practice change and implements it systematically. The key difference: QI uses routinely collected quality data and PDSA methodology; EBP requires a formal integrative or systematic literature review and an EBP framework (Iowa Model, PARIHS) as the guiding structure.

**Q: Does a DNP quality improvement project need IRB approval?**
Most DNP quality improvement projects qualify as QI non-research under 45 CFR 46 and require only a QI determination letter rather than full IRB review. The key criteria are: the project uses routinely collected clinical quality data, its purpose is to improve care at a specific site (not to contribute to generalizable knowledge), and the findings are not intended for publication as primary research. Students should submit a QI determination request to the IRB to obtain formal documentation rather than self-classifying.
