## EAV Depth Map
**Primary Entity**: DNP EBP Implementation Project

**Main Attributes**: EBP vs QI distinction, framework, literature synthesis, pilot design, staff education, outcome measurement, policy development

**Derived Attributes**:
- **EBP vs QI Distinction**: EBP implementation = formal synthesis of published evidence drives the intervention; QI = local quality data gap drives the improvement. EBP requires: (1) systematic/integrative literature review of 15–30+ studies, (2) explicit EBP framework application, (3) evidence synthesis table demonstrating the evidence base. QI does not require a formal literature synthesis — it starts from a local quality metric problem.
- **EBP Framework Requirement**: Iowa Model (most common — trigger identification, team assembly, evidence search, pilot, implement, evaluate, sustain), PARIHS/i-PARIHS (evidence × context × facilitation), Johns Hopkins PET, KTA, ARCC, ACE Star — one framework selected and explicitly applied to all project phases
- **Literature Synthesis Requirement**: minimum 15–30 primary peer-reviewed studies; integrative or systematic review methodology; evidence synthesis table (author/year/design/sample/intervention/outcomes/level of evidence/limitations); PRISMA flow diagram
- **Pilot Implementation Design**: unit or population-limited before site-wide rollout (e.g., implement in one unit for 6–8 weeks before expanding); pre-pilot staff education; fidelity assessment during pilot; adjust protocol based on pilot findings before full implementation
- **Staff Education Component**: content (what staff need to know and do differently — evidence base + protocol steps), format (in-person, e-learning, return demonstration), competency assessment (pre/post knowledge quiz, ≥80% pass threshold)
- **Policy/Protocol Document**: new clinical policy or procedure document is a required deliverable in most EBP implementation capstones — demonstrates that the practice change is institutionalised
- **Outcome Measurement**: pre-post comparison design (historical baseline vs post-implementation data); validated instruments where applicable; process and outcome measures
- **EBP Examples**: pressure injury prevention protocol (Wound Ostomy Continence Nurses — WOCN guidelines), hand hygiene improvement program (WHO multimodal hand hygiene strategy), pain assessment protocol (CPOT for non-verbal ICU patients), discharge education using teach-back method (PFSH guidelines)

#### H2: What Is a DNP Evidence-Based Practice Implementation Project?
> Define EBP implementation: systematic process of translating synthesised evidence from the published literature into a sustainable clinical practice change. Contrast with QI: EBP starts from evidence; QI starts from a local quality gap. DNP EBP project output: new clinical protocol or policy + evidence of outcome improvement + sustainability plan.

#### H2: Choosing an EBP Framework: Iowa Model, PARIHS, or Johns Hopkins?
> Iowa Model (most common for DNP EBP — starts from practice trigger, requires team assembly, evidence search and synthesis, pilot, evaluate, sustain — all phases documented in proposal and manuscript); PARIHS (best when implementation challenge is context or facilitation — complex organisational settings); Johns Hopkins PET (best when clinical site already uses Johns Hopkins EBP infrastructure). Must justify selection based on project type and setting.

#### H2: How Many Studies Do You Need for a DNP EBP Literature Review?
> 15–30 primary peer-reviewed studies is the standard minimum. More than 30 adds appraisal burden without quality gain. Integrative review most appropriate for most DNP EBP topics. Evidence synthesis table required. PRISMA flow diagram required. JBI or CASP appraisal tools for each included study. Evidence levels (Johns Hopkins Level I–V or GRADE certainty ratings).

#### H2: Designing the Practice Change Intervention: From Evidence to Protocol
> Steps: synthesise evidence → identify key intervention components → develop implementation protocol → create educational materials → pilot with small sample → evaluate fidelity → adjust → full implementation. Protocol specificity: the protocol document must be detailed enough for a nurse who has not participated in the training to follow independently.

#### H2: Staff Education as Part of Your DNP EBP Project
> Content (evidence base + protocol steps + documentation requirements), format (in-person for complex protocols; e-learning for large staff groups; one-page quick reference for reinforcement), timing (2–4 weeks before implementation), competency verification (pre-post quiz, return demonstration for skill-based changes). Education outcomes = process measures in the results section.

#### H2: Measuring Outcomes After EBP Implementation: Pre-Post Comparison Design
> Baseline measurement (4–8 weeks of historical data before implementation), post-implementation measurement (at end of implementation period — Week 8, 10, or 12). Paired t-test (continuous outcomes), McNemar (categorical). Process measures: staff education completion rate, protocol adherence rate. Outcome measures: clinical metric linked to PICOT O.

#### H2: DNP EBP Project vs Quality Improvement Project: Key Differences
> Comparison table: EBP (literature synthesis required; EBP framework; pilot phase; policy deliverable; 15–30+ studies) vs QI (quality data driven; PDSA/IHI methodology; multiple rapid cycles; existing data; literature review less formal). When a project is both (many DNP capstones have QI elements within an EBP structure — use the predominant methodology to classify).

### FAQ SECTION

**Q: How is an EBP implementation project different from a quality improvement project?**
An EBP implementation project is driven by a formal synthesis of published literature — the intervention is selected because the evidence supports it, and the project implements that evidence-based intervention in a clinical setting. A quality improvement project is driven by a local quality metric gap — the solution may be based on evidence, but the formal literature synthesis is not the primary driver. EBP projects require an evidence synthesis table with 15–30+ appraised studies; QI projects typically do not require this level of formal literature synthesis.

**Q: Does an EBP implementation DNP project need a pilot phase?**
A pilot phase is recommended but not universally required. Piloting the intervention on a smaller scale (one unit, one provider group, or a subset of patients) before full implementation allows fidelity assessment and protocol refinement. When a pilot is included, the proposal should describe both the pilot phase and the full implementation phase separately, with distinct timelines and evaluation plans. Some universities require a pilot phase as part of the EBP implementation methodology.
