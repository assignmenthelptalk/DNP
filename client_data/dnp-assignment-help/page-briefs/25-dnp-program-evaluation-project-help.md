## EAV Depth Map
**Primary Entity**: DNP Program Evaluation Capstone

**Main Attributes**: evaluation frameworks, evaluation questions, data sources, mixed methods design, IRB, dissemination, common tracks

**Derived Attributes**:
- **Evaluation Frameworks**:
  - **Logic Model**: inputs → activities → outputs → short-term outcomes → long-term outcomes → impact. Used to map program components and expected results. W.K. Kellogg Foundation Logic Model template.
  - **Donabedian Model**: Structure (resources, settings, qualifications) → Process (care delivery activities, protocol adherence) → Outcome (patient outcomes, quality indicators). Most commonly used in healthcare program evaluation.
  - **CIPP Model**: Context (needs, goals) → Input (resources, alternatives) → Process (implementation fidelity) → Product (outcomes, value). Stufflebeam (2003). Evaluative throughout all phases.
  - **RE-AIM Framework**: Reach (who was served) → Effectiveness (did it work?) → Adoption (who implemented it?) → Implementation (fidelity) → Maintenance (sustainability). Used for evaluating dissemination and sustainability.
- **Evaluation Questions**: formulated from the framework — must address structure, process, and outcome domains. Example: "Does the nurse residency program (structure) follow the established curriculum as designed (process), and does participation in the program result in improved RN retention rates and competency scores at 12 months (outcome)?"
- **Data Sources**: program records (attendance, completion rates, graduation records), patient satisfaction surveys (Press Ganey, CAHPS), clinical outcome databases (NDNQI, NHSN), staff competency assessments (pre/post scores), existing administrative data (turnover rates, certification rates), prospective surveys (knowledge, satisfaction, intent-to-stay)
- **Mixed Methods Design**: quantitative (program records, clinical outcomes data) + qualitative (focus groups, interviews, open-ended surveys) — provides comprehensive evaluation picture; common in education and leadership track program evaluations
- **IRB Considerations**: program records review (retrospective) — usually QI exempt; prospective surveys of program participants — expedited review; focus groups or interviews — expedited review; sensitive topics or vulnerable populations — full board may apply
- **Common Tracks**: Nurse Executive/Leadership (nurse residency program, staff development program), Population Health (community health program), Nursing Education faculty (curriculum evaluation), CNL (care delivery model evaluation)

#### H2: What Is a DNP Program Evaluation Capstone Project?
> Define program evaluation as systematic assessment of an existing clinical program, health education initiative, or care delivery model against defined goals and outcomes. Common in Nurse Executive, Population Health, CNL, and Nursing Education DNP tracks. Distinguishes from QI (measuring compliance with a process) and EBP (implementing a new practice): evaluation assesses whether an EXISTING program is meeting its goals.

#### H2: Choosing a Program Evaluation Framework: Logic Model vs Donabedian vs CIPP
> All four frameworks described with "when to choose" criteria: Logic Model (best for program planning + evaluation, new programs, grant-funded initiatives), Donabedian (best for healthcare quality program evaluation — maps directly to structure-process-outcome), CIPP (best for comprehensive formative + summative evaluation throughout the program lifecycle), RE-AIM (best when evaluating dissemination and sustainability potential). One framework selected; others acknowledged.

#### H2: Writing Evaluation Questions That Align With Program Goals
> Evaluation questions derived from the selected framework. Structure: one question per framework domain. Donabedian example: "Are the structural components of the nurse residency program adequately resourced (structure)? Is the curriculum delivered as designed (process)? What is the impact on 12-month RN retention rate (outcome)?" Evaluation questions ≠ research hypotheses — they are guiding assessment questions, not predictions.

#### H2: Data Sources for DNP Program Evaluation: Records, Surveys, and Interviews
> Types: retrospective (program records, administrative data — usually QI exempt), prospective quantitative (structured surveys, competency assessments — expedited IRB), prospective qualitative (interviews, focus groups — expedited IRB). Mixed methods is the most common design for DNP program evaluations. How to request access to existing program records.

#### H2: Mixed Methods in DNP Program Evaluation: Combining Numbers and Narratives
> Concurrent or sequential mixed methods design. Quantitative strand (program records, outcome data → statistical analysis), qualitative strand (interviews/focus groups → thematic analysis). Integration of findings: how does the qualitative data explain the quantitative results? This is the doctoral-level analytical contribution. NVivo for qualitative; SPSS or Excel for quantitative.

#### H2: IRB Requirements for Program Evaluation DNP Projects
> Retrospective record review (existing data, de-identified) → QI non-research or exempt. Prospective survey of staff or participants → expedited. Focus groups or interviews → expedited. Vulnerable populations (patients with serious illness, children, prisoners) → full board. Program operators must provide written permission for data access even when IRB exemption applies.

#### H2: Writing the Evaluation Report: Findings, Conclusions, and Recommendations
> Evaluation report structure: Executive Summary, Program Description, Evaluation Purpose, Methods, Findings (by evaluation question), Conclusions, Recommendations, Appendices. Recommendations must be specific and actionable — not "continue the program." Example: "Recommend increasing mentor-to-resident ratio from 1:3 to 1:2 to address the primary qualitative finding of insufficient individualised feedback."

### FAQ SECTION

**Q: What is the difference between a program evaluation and a quality improvement project?**
A program evaluation systematically assesses whether an existing program is meeting its stated goals — it evaluates program effectiveness, fidelity, and sustainability. A quality improvement project identifies a specific quality metric gap and implements a targeted change to address it. Program evaluation typically involves multiple data sources (records, surveys, interviews), uses a structured evaluation framework (Logic Model, Donabedian), and produces a formal evaluation report with recommendations. QI projects use PDSA methodology and focus on a single, measurable clinical metric.

**Q: What type of IRB review does a DNP program evaluation require?**
It depends on the data sources. If the evaluation uses only existing program records and de-identified administrative data, it typically qualifies as QI non-research or exempt from IRB review. If it includes prospective surveys of program participants or staff, expedited IRB review is typically required. If it includes focus groups or individual interviews, expedited review is required. If the program serves vulnerable populations, full board review may be required. The student should submit a determination request to the IRB regardless of anticipated review level.
