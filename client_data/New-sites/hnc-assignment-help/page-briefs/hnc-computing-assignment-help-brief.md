#### H2: HNC Computing Programming Assignment: Code Analysis and Written Evaluation
> **EAV to cover**: programming unit assessment format, Pass criteria (working code, documentation), Merit criteria (analysis of design choices, algorithm justification), Distinction criteria (critical evaluation of solution, alternative approaches, synthesis of design rationale)
> **Word count**: 300–380 words
> **Specific content**:
> - Programming unit at HNC Level 4 is assessed on both the technical quality of the code and the analytical quality of the written evaluation
> - Pass criteria: produce a working program that meets the functional specification; include appropriate code documentation (comments, variable naming conventions, modular structure)
> - Merit criteria: analyse design decisions — justify the choice of data structures (arrays vs linked lists, for example), explain algorithm selection, compare the chosen approach to at least one alternative; the analysis must be specific to the code produced, not generic programming theory
> - Distinction criteria: critically evaluate the programming solution — assess its scalability, maintainability, and efficiency; evaluate alternative design patterns or architectural approaches (procedural vs object-oriented, for example); synthesise the technical decisions into a coherent design rationale with reference to software engineering principles
> - Written evaluation format: typically 800–1,500 words of analytical commentary accompanying the code submission; some assignments require a separate technical report
> - Common student error: submitting code with only inline comments and no analytical evaluation — this meets Pass documentation requirements but does not address Merit or Distinction criteria at all
> - Academic references: students should reference software engineering principles (e.g., SOLID principles, clean code practices) using academic Computing texts — not Stack Overflow posts

> **Specific content**:

> - Merit criteria: normalise to 3NF and analyse the design decisions — justify why specific attributes were placed in specific tables; explain referential integrity and why it matters for the scenario; compare normalised and unnormalised versions

### FAQ SECTION

**Q: What is the difference between the practical and written components of an HNC Computing assignment?**
HNC Computing assignments typically have two components: a practical output (working code, network diagram, ER diagram, UML model) and a written analytical component (evaluation report, design justification, analytical commentary). Pass criteria are often met by the practical output alone. Merit and Distinction criteria are almost exclusively located in the written analytical component — the student must justify design decisions, compare alternatives, and critically evaluate the solution. Many students lose Merit and Distinction grades by submitting strong practical work with minimal written analysis.

**Q: Do I need to reference sources in HNC Computing assignments?**
Harvard referencing is required for the analytical and theoretical sections of HNC Computing assignments. When defining networking protocols, evaluating development methodologies, or justifying a programming approach, academic sources should be cited — textbooks such as Tanenbaum's Computer Networks for networking or Sommerville's Software Engineering for systems analysis. Technical implementations (the code itself, network diagrams) do not require academic references, but the written justification of design decisions does.

**Q: How long are the written sections of HNC Computing assignments?**
Written evaluation and analytical commentary sections vary by unit. Programming unit evaluations are typically 800–1,500 words. Networking and Database Design reports are typically 1,500–2,500 words. Systems Analysis assignments often require 2,000–3,000 words of written analysis alongside UML models. The exact word count is stated in the Pearson assessment brief for each unit — students should check their specific brief.

**Q: Why do technically strong students sometimes get a Pass instead of Merit or Distinction in HNC Computing?**
Technical proficiency satisfies Pass criteria in most HNC Computing units — a working program, a correctly normalised database, or a functioning network design demonstrates the required knowledge at Pass level. Merit and Distinction criteria require a different cognitive output: analytical commentary on why design choices were made, comparison of alternatives, and critical evaluation of the solution's limitations and strengths. Students who come from IT roles or BTEC Level 3 Computing where technical output was the primary measure of quality often underestimate the written analytical requirement at Level 4.
