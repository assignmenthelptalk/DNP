### H2: Mobile-First Website Design for Kenya's Mobile-Dominant Market

**Contextual directive:** Cover mobile-first as a non-optional standard for Kenya. Use data. Connect to Google Mobile-First Indexing. This section bridges to the Mobile-First Website Design Kenya page via supplementary content.

**EAV depth required:**
- Kenya mobile stat: 80%+ of Kenyan internet users access via mobile (Safaricom 4G network dominant)
- Devices: Tecno Spark, Infinix Hot series, Samsung A-series — budget Android as primary design target
- Design implications: minimum 44px touch targets, 16px minimum font, single-column layouts, compressed images (WebP format), no hover-dependent navigation
- Google's Mobile-First Indexing: Google indexes the mobile version first — a desktop-only design is invisible to the algorithm
- Performance target: LCP under 2.5 seconds on 4G connection

### H2: Kenya-Specific Website Design Elements: M-Pesa, Swahili, and Low Bandwidth

**Contextual directive:** This section covers Kenya-specific design factors that generic web design guides miss. This is high-differentiation content — competitors in other markets cannot write this authentically.

**EAV depth required:**
- M-Pesa integration: Daraja API paybill number display, Lipa Na M-Pesa badge, STK push button — mandatory for any business selling online

- KRA compliance: if the website sells products, KRA VAT registration number must be visible
