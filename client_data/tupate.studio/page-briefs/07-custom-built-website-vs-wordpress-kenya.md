### H2: What "Custom-Built Website" Means for a Kenyan Business

**Contextual directive:** Define custom-built clearly before any comparison. Many Kenyan business owners have never encountered this approach — they assume all websites are built on WordPress or Wix.

**EAV depth required:**
- Entity: Custom-Built Website
- Definition: a website built entirely from HTML, CSS, and JavaScript — no CMS platform, no page builders, no plugin ecosystem. Every line of code serves a specific purpose.
- What it includes: hand-written semantic HTML (or component-based React/Next.js for larger projects), CSS written specifically for the site's design, vanilla JavaScript or minimal frameworks for interactivity
- What it does NOT include: WordPress, Elementor, WooCommerce, Wix, Squarespace, Shopify (unless specifically required for e-commerce), pre-made themes, visual page builders
- Analogy for Kenyan business owners: think of it as the difference between a custom-tailored suit from a Nairobi tailor (cut exactly for your body, using the fabric you choose, at the dimensions you need) vs buying a suit off a rack (standard sizes, standard fabric, not quite right but functional). Both work. One fits better.
- Who builds custom websites: a developer who writes code, not someone who installs and configures plugins. Tupate Studio's process is entirely code-based.

### H2: Why WordPress Websites Are Slower for Kenyan Mobile Users

**Contextual directive:** Performance is the most important comparison dimension for Kenya. Lead with hard data. This section must use specific numbers, not adjectives.

**EAV depth required:**
- Page weight comparison:
  - Average WordPress page: 2–4MB total page size (images + HTML + CSS + JavaScript combined)
  - Average custom-built page: 200–600KB total page size
  - On Safaricom 4G (average 15Mbps): 3MB page = ~1.6 seconds download; 500KB page = ~0.27 seconds download. Before rendering begins.
- Core Web Vitals comparison (real-world averages):
  - WordPress (Elementor theme): LCP 4.8s (FAIL), CLS 0.25 (FAIL), INP 380ms (FAIL) — typical unoptimized Kenyan WordPress site
  - Custom-built (Tupate Studio): LCP 1.4s (PASS), CLS 0.02 (PASS), INP 80ms (PASS) — typical result
  - Google PageSpeed Insights mobile scores: WordPress average 35–55; Custom-built average 88–98
- Why WordPress is heavy: plugin loading overhead (each active plugin adds CSS + JS), theme framework CSS (often 200KB+ just for styling), render-blocking scripts from plugins, large DOM size from page builder markup
- Kenya 3G scenario: rural Kenyan users on 3G (3–8Mbps) — a 3MB WordPress page can take 3–8 seconds. Abandonment rate at 3 seconds: 53% (Google data). A 500KB custom page loads in under 1 second on 3G.
- Speed as SEO signal: Google uses Core Web Vitals as a direct ranking factor since June 2021. A WordPress site failing Core Web Vitals ranks lower than a custom-built site passing them — all other factors equal.

### H2: Security: Why Custom-Coded Sites Face Fewer Threats Than WordPress in Kenya

**Contextual directive:** Security is a major pain point for Kenyan businesses — WordPress hacking is extremely common. This section validates a real fear and offers the solution.

**EAV depth required:**
- WordPress attack surface: 43% of websites on WordPress = the largest single target for automated hacking tools
  - Common attack vectors targeting Kenyan WordPress sites: brute-force login attacks on `/wp-admin` URL (universal WordPress login path); plugin vulnerabilities (CVEs published publicly — hackers automate exploitation); theme vulnerabilities; xmlrpc.php exploits; SQL injection via vulnerable form plugins
  - Real impact for Kenyan businesses: website defacement (hacker replaces content with their own), SEO spam injection (spammy pharma/gambling links injected into pages — destroys Google rankings), data theft from contact form databases, ransomware that encrypts the server
- Custom-built attack surface: no standard attack paths. There is no `/wp-admin` URL to brute-force. No plugins with published CVEs. No standard database schema to exploit. An attacker cannot use automated tools — they must manually probe each custom site individually, which is not economically viable for opportunistic hackers.
- Maintenance comparison:
  - WordPress: WordPress core updates (2–4 times/year), plugin updates (weekly — each update can break other plugins), theme updates, PHP version upgrades — each requiring testing and sometimes paid developer intervention
  - Custom-built: updates only when functionality changes or new features are added. Security patches applied to custom code when a specific vulnerability is identified (rare, not automated hacking targets).
- Kenya-specific context: Kenyan WordPress sites are targeted heavily because:
  1. Many are on cheap shared hosting with outdated PHP versions
  2. Many have plugins that haven't been updated in 1–3 years (owners don't know how)
  3. WordPress's predictable URL structure makes mass automated attacks efficient

  - 3-year total: Ksh 180,000–200,000 for a maintained, secure WordPress site
  - Note: many Kenyan business owners spend Ksh 8,000–15,000 on a WordPress site and then spend nothing on maintenance — they wonder why it gets hacked and ranks nowhere.

### H2: What Kenyan Businesses Get With a Custom-Built Website That WordPress Can't Deliver

**Contextual directive:** Cover the positive case — what custom unlocks, not just what WordPress lacks.

**EAV depth required:**
- Exact functionality: every feature is built for the business's specific need. No compromise for plugin limitations.
  - Example: a Nairobi industrial supplier needs a filterable product catalog by machine type + available parts + compatible brands. WordPress plugin for this: Ksh 15,000/year; often breaks. Custom-built solution: part of the initial build, works exactly as needed.
- Clean semantic HTML: written with proper heading hierarchy, schema-ready markup, no conflicting plugin SEO settings — provides a structural SEO advantage from day one
- No conflicting SEO plugins: WordPress sites often have Yoast AND RankMath AND All in One SEO installed simultaneously (causing conflicts and duplicate meta tags). Custom sites have one SEO approach, implemented precisely.
- Full code ownership: the client owns every line of code. No license fees. No platform lock-in. If Tupate Studio ceased to exist, any competent developer could continue the work.
- Kenya-specific customizations: M-Pesa STK Push, WhatsApp business button, Swahili/English language toggle — built exactly as needed, not adapted from a plugin that was designed for PayPal and UK businesses.
- Performance guarantee: Tupate Studio builds to a minimum 85/100 PageSpeed Insights mobile score. No WordPress site with a full plugin stack reliably achieves this without expensive optimization work.
