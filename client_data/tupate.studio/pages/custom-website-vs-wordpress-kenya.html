Security: Why Custom-Coded Sites Face Fewer Threats Than WordPress in Kenya

WordPress websites are the primary target of automated website hacking tools globally — and Kenyan WordPress websites are attacked with the same automated tools used worldwide, at the same frequency, with the same exploits. Understanding why requires understanding how WordPress's attack surface differs from a custom-built site's attack surface.

The WordPress attack surface: 43% of all websites running on WordPress means that a single automated hacking tool can attempt the same attack — against the same URL path, using the same exploit — on hundreds of millions of websites simultaneously. The attack vectors most commonly used against Kenyan WordPress sites are not sophisticated hacking operations. They are automated scripts running against known vulnerabilities in predictable locations:

Brute-force login attacks target `/wp-admin` — the universal WordPress login URL that is the same on every WordPress installation worldwide. Automated bots attempt thousands of username/password combinations against this URL every day. Plugin vulnerability exploits use publicly published CVE (Common Vulnerabilities and Exposures) records to attack specific plugin versions — hackers scan for sites running a vulnerable version and exploit it automatically before the site owner knows the update exists. The xmlrpc.php file, present on all WordPress installations, provides an XML-based API that is frequently exploited for credential stuffing and DDoS amplification attacks even when the site owner has never used XML-RPC intentionally.

Real consequences for Kenyan businesses: Website defacement, where a hacker replaces your homepage content with their own message or imagery — a common Kenyan experience. SEO spam injection, where pharmaceutical or gambling keyword spam is injected into your pages, destroying your Google rankings and triggering Google's malware warning for visitors. Data theft from contact form and customer databases stored on the WordPress server. Ransomware deployment that encrypts the entire hosting server. These are not theoretical risks — they are common occurrences on Kenyan WordPress sites, particularly those on shared hosting with outdated PHP versions and unpatched plugins.

The custom-built attack surface: A custom-built website has no standard attack paths. There is no `/wp-admin` URL to brute-force. There are no plugins with published CVEs to exploit automatically. There is no standard database schema that hacking tools know how to probe. An attacker cannot use automated tools against a custom-built website — they must manually probe each site individually for site-specific vulnerabilities, which is not economically viable for opportunistic attackers who rely entirely on automated exploitation of known targets. Custom sites are boring to automated hacking tools because they offer no predictable entry points.

Maintenance comparison: WordPress requires WordPress core updates (2–4 times per year), plugin updates (typically weekly — each update can introduce incompatibilities with other plugins, requiring testing and sometimes developer intervention to resolve), theme updates, and PHP version upgrades on the hosting server. Each of these is a recurring overhead of time or money for the site owner. Custom-built websites receive updates only when functionality changes or new features are added. There are no third-party plugin update cycles to track.

Kenya-specific context for why WordPress hacking is especially common: the majority of Kenyan WordPress sites run on cheap shared hosting with outdated PHP versions (PHP 7.x when 8.x is current), because the hosting provider does not upgrade automatically and the site owner does not know they need to request the upgrade. Many have plugins that have not received updates in 1–3 years because the owner does not know how to update them, or the plugin was abandoned by its developer. WordPress's universal URL structure makes mass automated attacks economically efficient in a way that no custom-built site architecture can be targeted.
